# مشروع Flask مع قالب HTML

هذه المستودع يحول الصفحة `index.html` الموجودة إلى تطبيق Flask بسيط بدون تغيير التصميم.

## كيفية الاستخدام

1. أنشئ بيئة افتراضية:
   ```bash
   python -m venv venv
   ```
2. فعِّل البيئة (في ويندوز):
   ```bash
   venv\\Scripts\\activate
   ```
3. ثبِّت المتطلبات:
   ```bash
   pip install -r requirements.txt
   ```
4. شغل التطبيق:
   ```bash
   python app.py
   ```
5. افتح المتصفح واذهب إلى `http://127.0.0.1:5000/`.

### صفحة المناهج والمحتوى

- بعد تشغيل التطبيق، ستجد رابط "المنهج الدراسي" في الصفحة الرئيسية يوجّهك إلى `/curriculum`.
- يتم قراءة الملفات من المجلد `b`، ويقوم الخادم ببناء هيكلية متداخلة تتبع الشجرة الفعلية للمجلدات. هذا يعني أن أي ترتيب تستخدمه في `b` (مثلاً التصنيف ثم الفصل أو الفصل ثم التصنيف) سيُحترم تلقائياً.
- يوجد قسم افتراضي `الخيرات` مضاف تلقائياً داخل الصفحة، ويمكنك اختيار السنة ليعرض شروحات الفيديو المُضَمَّنة. يمكنك تعديل هذه الروابط مباشرة عبر تغيير القاموس `videos_map` في `app.py` أو حذف القسم إذا لم تُرِد استخدامه.
- يُستخدم SQLite لتخزين معلومات الملفات (مساراتها، القسم، المادة، الفصل) بحيث لا يتم مسح كل مرة.
- إذا قمت بإضافة ملفات جديدة في المجلد `b`، فلن تحتاج إلى فعل شيء إضافي: المجلد يُفحص ويعاد بناء قاعدة البيانات في كل مرة تدخل فيها صفحة `/curriculum` أو عند استدعاء المسار الخاص `/refresh`.

### تحجيم المحتوى

- بدلاً من التعامل مع ملفات HTML ضخمة، يتم تخزين البيانات في قاعدة SQLite (`curriculum.db`).
- الجدول `files` يحتوي الأعمدة: `dept`,`subject`,`category`,`semester`,`name`,`relpath`.
- يمكن تعديل أو توسيع التطبيق لتستخدم واجهة لإدارة المحتوى أو قاعدة بيانات أوسع فيما بعد.


الصفحة تظهر نفس التصميم الموجود في `index.html` دون أي تغييرات.

> الصور في قسم "الروابط المهمة" تُحمّل من المجلد `a` الذي يُخدم ثابتاً عبر Flask (يُتاح الوصول إليها من `/a/1.jpg` وهكذا).
